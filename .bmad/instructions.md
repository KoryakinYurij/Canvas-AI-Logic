# BMAD Method — Инструкция для Jules (v1.0)

## Описание
BMAD Method — это специфицированная система многоагентной разработки, в которой каждый этап проекта выполняет специализированный агент, а управление процессом ведёт оркестратор. Все инструкции и шаблоны хранятся внутри директории `.bmad/`.

---

## Процесс и последовательность действий

**1. Инициация сессии**
- Прочитай текущий файл `.bmad/instructions.md`
- Определи тип проекта: Greenfield, Brownfield, Service-only, UI-focused, в соответствии с запросом пользователя.
- Сравни входные данные с шаблонами воркфлоу в `.bmad/workflows/`, выбери подходящий.

**2. Запуск главного агента (Orchestrator / Master)**
- Инициируй агент-оркестратор, который определяет последовательность запуска агентов согласно выбранному воркфлоу.

**3. Вызов и запуск агентов**
- Для каждого предусмотренного этапа:
    - Инициализируй специализированного агента, загрузив его сценарий из `.bmad/agents/agent-<роль>.md`
    - Следуй чек-листу: получи необходимые входные артефакты, подготовь выходной документ.
    - Сохрани артефакт в соответствующей директории (например, `project-brief.md`, `architecture.md`, эпики, user stories и т.д.).
    - Передай результат следующему агенту.
- Если обнаружен разрыв или недостаток — инициируй возврат к предыдущему агенту/этапу.

**4. Управление воркфлоу**
- Работай строго по описанию выбранного воркфлоу (см. `.bmad/workflows/`).
- В Greenfield: последовательность — Analyst → PM → Architect → PO → SM → Dev → QA.
- В Brownfield: Analyst → Architect → SM → Dev.

**5. Эмулирование агентов**
- Каждый агент действует по описанию в файле `.bmad/agents/agent-<роль>.md`, использует шаблоны output-ов, чек-листы и sample-outputs.
- Jules обязана эмулировать поведения агентов, обеспечивать автогенерацию документов через markdown/YAML шаблоны.
- Сохраняй историю коммуникаций и итоговые артефакты в `.bmad/history/` для повторного анализа.

---

## Применение инструкций в новой сессии

1. Начни всегда с анализа (или запроса пользователю — для оркестрации).
2. Запусти мастера/оркестратора, следуя описанию воркфлоу.
3. Активируй каждого агента по очереди, инициализируй с помощью соответствующего файла.
4. Генерируй документы, тесты, справки, веди changelog.
5. Возвращай процесс на этап назад, если не хватает информации.
6. Всегда используй чек-листы и шаблоны в `.bmad/agents/` и `.bmad/workflows/`.

---

## Структура .bmad/

- instructions.md — этот файл
- agents/ — сценарии и описание ролей агентов
- workflows/ — воркфлоу по типу проектов, шаблоны документов
- history/ — сохранение истории взаимодействий
- docs/ — создаваемые артефакты (PRD, архитектура, user-stories и др.)

---

## Пример команд и последовательности

- `npx bmad-method install` — стартовая инициализация
- `*analyst` — запуск анализа требований
- `*pm` — составление PRD
- `*architect` — описания архитектуры
- `*po` — проверка целостности
- `*sm` — детализация историй
- `*dev` — разработка и автогенерация кода
- `*qa` — финальное тестирование

---

## Требования

- Jules должна автоматически определять и запускать агентов на каждом этапе
- Эмулировать ролевые сценарии через markdown/YAML-файлы
- Работать строго по структуре бэкап-файлов и истории
- Итог работы — комплект документов и changelog для контроля версий

---

## Чек-лист агента-оркестратора для старта проекта

- Проверить наличие `.bmad/agents/`, `.bmad/workflows/`
- Определить тип воркфлоу
- Запустить последовательность агентов
- Сохранять все артефакты и историю изменений
